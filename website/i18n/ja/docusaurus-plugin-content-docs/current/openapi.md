# OpenAPI と型生成

Koya はコントローラーから型安全なクライアント型（`AppType`）と OpenAPI 3.1 仕様を自動生成します。デコレーターやアノテーションは不要です。

## 概要

`@koya/contract` パッケージは以下を提供します:

- **AppType** — Hono の `hc` クライアント用 TypeScript 型（完全な型安全 API 呼び出しを実現）
- **OpenAPI 3.1** — コントローラーのシグネチャから生成される標準 API ドキュメント

## インストール

```bash
pnpm add @koya/contract
```

## 設定

プロジェクトルートに `koya.config.ts` を作成します:

```typescript
import { defineConfig } from '@koya/contract';

import { HelloController } from './src/controllers/hello.controller';
import { UsersController } from './src/controllers/users.controller';

export default defineConfig({
  controllers: [
    { class: HelloController, source: './src/controllers/hello.controller.ts' },
    { class: UsersController, source: './src/controllers/users.controller.ts' },
  ],
  dist: './generated',
  tsconfig: './tsconfig.json',
});
```

### 設定オプション

| オプション | 型 | 説明 |
|--------|------|-------------|
| `controllers` | `ControllerEntry[]` | ソースパス付きのコントローラーリスト |
| `dist` | `string` | 生成ファイルの出力ディレクトリ |
| `tsconfig` | `string` | tsconfig.json へのパス（OpenAPI 生成に必須） |

## ファイル生成

### 一度だけビルド

```bash
pnpm koya-contract build
```

以下が生成されます:
- `<dist>/app.gen.ts` — Hono クライアント用 AppType
- `<dist>/openapi.json` — OpenAPI 3.1 仕様

### ウォッチモード

```bash
pnpm koya-contract watch
```

コントローラーの変更時に継続的にファイルを再生成します。

### npm スクリプト

`package.json` に追加:

```json
{
  "scripts": {
    "generate": "koya-contract build",
    "generate:watch": "koya-contract watch"
  }
}
```

## 生成されるファイル

### app.gen.ts

生成された `app.gen.ts` は Hono の `hc` クライアントで使用できる `AppType` をエクスポートします:

```typescript
// THIS FILE IS GENERATED BY @koya/contract. DO NOT EDIT.
import type { Route, BuildAppType } from '@koya/contract';
import type { HelloController } from '../src/controllers/hello.controller';

export type AppType = BuildAppType<[
  Route<'GET', '/hello/:name', typeof HelloController.prototype.greet>,
  Route<'POST', '/hello', typeof HelloController.prototype.create>,
]>;
```

### openapi.json

標準の OpenAPI 3.1 仕様:

```json
{
  "openapi": "3.1.0",
  "info": {
    "title": "koya app",
    "version": "0.0.0"
  },
  "paths": {
    "/hello/{name}": {
      "get": {
        "parameters": [...],
        "responses": {...}
      }
    }
  },
  "components": {
    "schemas": {...}
  }
}
```

## AppType の使い方

### 型安全な API クライアント

```typescript
import { hc } from 'hono/client';
import type { AppType } from './generated/app.gen';

const client = hc<AppType>('https://api.example.com');

// 完全な型付け - IDE の自動補完と型チェック
const response = await client.hello[':name'].$get({
  param: { name: 'world' },
});

if (response.ok) {
  const data = await response.json();
  // data は { message: string } として型付けされる
  console.log(data.message);
}
```

### 型安全なクライアントでのテスト

```typescript
import { hc } from 'hono/client';
import { describe, it, expect } from 'vitest';
import { app } from './app';
import type { AppType } from './generated/app.gen';

describe('Hello API', () => {
  const client = hc<AppType>('http://localhost', {
    fetch: (input, init) => app.fetch(new Request(input, init)),
  });

  it('should return greeting', async () => {
    const res = await client.hello[':name'].$get({
      param: { name: 'world' },
    });

    expect(res.status).toBe(200);
    const body = await res.json();
    expect(body.message).toBe('Hello, world!');
  });
});
```

## 仕組み

Koya は [Scramble](https://scramble.dedoc.co/) にインスパイアされた「ゼロアノテーション」アプローチを採用しています:

1. **静的解析** — ビルド時にコントローラーメソッドのシグネチャを解析
2. **型抽出** — TypeScript の型からリクエスト/レスポンス型を抽出
3. **コード生成** — 型レベル計算を使用して `AppType` を生成
4. **スキーマ生成** — TypeScript 型を OpenAPI 用 JSON Schema に変換

これにより、ランタイムコードはクリーンなまま保たれます。バリデーションのために書いた定義以外に、デコレーターやスキーマ定義は不要です。
