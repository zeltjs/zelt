# Integration Tests

Integration tests following NestJS pattern - each feature has its own mini-app and e2e tests.

## Structure

```text
integration/
├── hello-world/           # Basic HTTP test
│   ├── src/              # App code
│   │   ├── app.ts
│   │   └── hello.controller.ts
│   └── e2e/              # E2E tests
│       └── hello-world.spec.ts
├── scripts/
│   ├── switch-mode.sh    # Switch between dist/pack/public modes
│   └── run-tests.sh      # Run integration tests
└── README.md
```

## Usage

```bash
# Run tests in dist mode (default, local built packages)
./integration/scripts/run-tests.sh dist

# Run tests in pack mode (npm-packed tarballs)
./integration/scripts/run-tests.sh pack

# Run tests in public mode (npm registry versions)
./integration/scripts/run-tests.sh public

# Run tests in all modes
./integration/scripts/run-tests.sh all

# Run specific test
./integration/scripts/run-tests.sh dist hello-world
```

## Modes

| Mode | Description | Use Case |
|------|-------------|----------|
| `dist` | Uses local built packages (`file:../../packages/*`) | Development |
| `pack` | Uses npm-packed tarballs (`file:../.pack-cache/*.tgz`) | Pre-release validation |
| `public` | Uses npm registry versions | Regression testing |
| `all` | Runs tests in all modes sequentially | Full validation |

## Adding new tests

1. Create a new directory under `integration/`
2. Add `tsconfig.json`, `vitest.config.ts`
3. Create `src/` for app code and `e2e/` for tests
4. Use `@zeltjs/testing` for test utilities
5. `package.json` is auto-generated by `switch-mode.sh`
