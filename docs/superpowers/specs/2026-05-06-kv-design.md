# @zeltjs/kv Design Spec

## Overview

Edge/serverless 環境向けの key-value store 抽象パッケージ。capability 別 interface (`KVStore` / `AtomicKVStore`) で driver 互換性の嘘を排除し、`namespace` を最上段で必須化することで use-case 間のキー空間衝突を構造的に防止する。reference 実装として `MemoryKV` を同梱し、dev/test の zero-config を成立させる。

## Goals

- driver 横断の最小 API（`get` / `set` / `del` / `has` / `expire` / `namespace`）
- atomic capability の型レベル分離（`AtomicKVStore extends KVStore`）— CF KV のような eventually consistent な driver を rate-limit に渡すと compile error
- namespace 必須化によるキー空間衝突の構造的排除（最上段の `KVDriver` は data ops を持たない）
- JSON 自動シリアライズ（driver 差吸収）
- 全 driver 共通のコンプライアンステスト suite を `@zeltjs/kv/testing` subpath で提供

## Non-Goals

- pub/sub（別パッケージ `@zeltjs/pubsub` で将来）
- object/file storage（`@zeltjs/storage` で別途、cache より後）
- batch ops (`mget` / `mset`)（v1 では含めない、必要があれば v2）
- key の列挙 / scan（CF KV / Redis で本番危険、API 化しない）
- tag invalidation（`@zeltjs/cache` の責務）

## API Design

### Interfaces

```typescript
/** driver の最上段。namespace で view を取り出す。data ops は持たない */
export interface KVDriver {
  namespace(prefix: string): KVStore;
}

/** atomic 操作対応版 driver */
export interface AtomicKVDriver extends KVDriver {
  namespace(prefix: string): AtomicKVStore;
}

/** namespaced view。実際の data ops はここ */
export interface KVStore {
  get<T>(key: string): Promise<T | undefined>;
  set<T>(key: string, value: T, opts?: { ttlSec?: number }): Promise<void>;
  del(key: string): Promise<void>;
  has(key: string): Promise<boolean>;
  /** TTL 延長 (session touch / lock extend 用) */
  expire(key: string, ttlSec: number): Promise<boolean>;
  /** 子 namespace。チェーン可能 */
  namespace(prefix: string): KVStore;
}

/** atomic 操作対応 view */
export interface AtomicKVStore extends KVStore {
  /** atomic incr。最初の incr 時に同時に TTL をセット可能 (race-free) */
  incr(key: string, by?: number, opts?: { ttlSec?: number }): Promise<number>;
  /** atomic set if not exists。lock / idempotency 用 */
  setnx<T>(key: string, value: T, opts?: { ttlSec?: number }): Promise<boolean>;
  /** 値一致時のみ削除。lock release で他人の lock を消さない保証 */
  delIf(key: string, expected: unknown): Promise<boolean>;
  namespace(prefix: string): AtomicKVStore;
}
```

### MemoryKV (reference impl)

```typescript
@Injectable()
export class MemoryKV implements AtomicKVDriver {
  namespace(prefix: string): AtomicKVStore { /* ... */ }
}
```

| 実装方針 | 内容 |
|---|---|
| ストレージ | `Map<string, { value: string; expiresAt?: number }>` |
| TTL 削除 | 遅延削除（get 時にチェック）+ 軽量 GC タイマー |
| atomic ops | 単一プロセス前提の in-memory 直列実行 |
| マルチプロセス | 非対応（Redis driver を使うこと） |
| シリアライズ | 内部で JSON 文字列保持（Redis と挙動を揃えるため） |

### namespace 規則

| ルール | 内容 |
|---|---|
| 最上段で data ops 不可 | `KVDriver` は `namespace` のみを持つ。型レベルで `kv.get(...)` を不可能にする |
| 空 prefix 禁止 | `kv.namespace('')` は `MinPrefixLengthError` を throw |
| 区切り文字 | driver タッチしない。user が `'cache:'` `'cache/'` `'cache.'` 等を任意指定 |
| チェーン可能 | `kv.namespace('a:').namespace('b:')` → 内部 prefix は `'a:b:'` |

### JSON シリアライズ

- `set(key, value)` 時に `JSON.stringify(value)` を内部で実行
- `get<T>(key)` 時に `JSON.parse` を実行、型は caller の `<T>` で assert
- `undefined` value は `set` 不可（throw）
- バイナリ値の取り扱いは v2 で `setRaw` / `getRaw` を検討

### TTL

- 単位: 秒（`ttlSec: number`）
- driver 制約は driver 側で `MinTtlError` として throw（CF KV: 60s 未満は不可など）
- TTL 未指定の `set` は永続化（driver 設定の最大保持時間に従う）

### Errors

```typescript
export class KVError extends Error {}
export class UnsupportedOperationError extends KVError {}
export class MinTtlError extends KVError {}
export class MinPrefixLengthError extends KVError {}
```

## Compliance Test Suite

```typescript
// @zeltjs/kv/testing
export const runKVStoreComplianceTests = (factory: () => KVDriver) => {
  describe('KVStore compliance', () => {
    let kv: KVStore;
    beforeEach(() => { kv = factory().namespace('test:'); });

    it('get returns undefined for missing key');
    it('set + get round-trips JSON object');
    it('TTL expires the key');
    it('expire(key) extends/sets TTL');
    it('has reflects deletion');
    it('namespace isolates keys (cache:foo vs ratelimit:foo)');
    it('chained namespace concatenates prefixes');
    it('empty prefix throws MinPrefixLengthError');
  });
};

export const runAtomicKVStoreComplianceTests = (factory: () => AtomicKVDriver) => {
  runKVStoreComplianceTests(factory);
  describe('AtomicKVStore compliance', () => {
    let kv: AtomicKVStore;
    beforeEach(() => { kv = factory().namespace('test:'); });

    it('incr is atomic under concurrent calls');
    it('incr with ttlSec sets TTL only on first incr (no extension on subsequent)');
    it('setnx returns false on existing key, true on new key');
    it('delIf only deletes when value matches expected');
  });
};
```

各 driver はこの suite を自身のテストで実行する:

```typescript
// @zeltjs/kv-driver-redis/test/compliance.test.ts
import { runAtomicKVStoreComplianceTests } from '@zeltjs/kv/testing';
import { RedisKV } from '../src';

runAtomicKVStoreComplianceTests(() => new RedisKV(testRedisConfig));
```

## Usage Example

### Basic Setup (dev/test, MemoryKV)

```typescript
// src/configs/cache.config.ts
import { Config, inject } from '@zeltjs/core';
import { MemoryKV, type KVStore } from '@zeltjs/kv';

@Config
export class CacheConfig {
  static readonly Token = CacheConfig;

  constructor(kv = inject(MemoryKV)) {
    this.store = kv.namespace('cache:');
  }

  readonly store: KVStore;
  defaultTtlSec = 300;
}
```

### Type-level Capability Check

```typescript
// CloudflareKV は KVDriver のみ実装 (eventual consistency のため)
@Config
export class RateLimitConfig {
  static readonly Token = RateLimitConfig;

  constructor(kv = inject(CloudflareKV)) {
    this.store = kv.namespace('rate-limit:');
    //          ~~~~~~~~~~~~~~~~~~~~~~~~~~~ Type 'KVStore' is not assignable to 'AtomicKVStore'
  }

  readonly store: AtomicKVStore;
}
```

### namespace チェーン

```typescript
const cacheStore = kv.namespace('cache:');
const userCache = cacheStore.namespace('user:');
await userCache.set('42', { name: 'Alice' });
// → 実キー: 'cache:user:42'
```

### 直接インスタンス化（テスト用途）

```typescript
import { MemoryKV } from '@zeltjs/kv';

const kv = new MemoryKV();
const store = kv.namespace('test:');
await store.set('foo', { bar: 1 });
expect(await store.get('foo')).toEqual({ bar: 1 });
```

## Error Handling

| Scenario | Behavior |
|---|---|
| Empty namespace prefix | `MinPrefixLengthError` を throw |
| TTL が driver 最小値未満 | `MinTtlError` を throw（driver 実装側） |
| `set(key, undefined)` | `KVError` を throw |
| atomic op が driver 未対応 | `UnsupportedOperationError`（型で防止できれば runtime にこない） |

## Testing Strategy

- `MemoryKV` 自身に `runAtomicKVStoreComplianceTests` を適用（spec の self-test）
- TTL タイマーのテスト（fake timers）
- namespace チェーンの prefix 結合テスト
- JSON シリアライズの edge case（`null`, `Date` 文字列化, 配列, ネスト）
- atomic 並行実行テスト（`Promise.all` で 100 並列 incr → 値が 100 になる）

## Future Considerations

- `mget` / `mset` batch ops — performance 用途で v2
- `setRaw` / `getRaw` — バイナリ値・大きい blob 向け、必要が出たら追加
- `cas` (compare-and-swap) — atomic interface に追加検討（現状 `delIf` で代替）
- v2 driver: `@zeltjs/kv-driver-upstash`, `@zeltjs/kv-driver-cloudflare`, `@zeltjs/kv-driver-durable-object`
- `@zeltjs/pubsub` — Redis pub/sub と CF DO broadcast を統合する別パッケージ
- TTL の `ms` 精度 — Redis `PEXPIRE` 等を使う場合の API 拡張
