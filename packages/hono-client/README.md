# @zeltjs/hono-client

[![Documentation](https://img.shields.io/badge/docs-zeltjs.com-blue)](https://zeltjs.com)

Generate type-safe Hono client types from Zelt controllers.

**[Read the Documentation](https://zeltjs.com)**

## Installation

```bash
npm install -D @zeltjs/hono-client
```

## Usage

### CLI

```bash
npx zelt-hono-client generate --config zelt.config.ts
```

### Plugin

```typescript
// zelt.config.ts
import { defineConfig } from '@zeltjs/cli';
import { honoClientPlugin } from '@zeltjs/hono-client';

export default defineConfig({
  plugins: [
    honoClientPlugin({
      output: './src/client.ts',
    }),
  ],
});
```

### Client

Use the generated types with Hono's `hc`:

```typescript
import { hc } from 'hono/client';
import type { AppType } from './client'; // generated by @zeltjs/hono-client

const client = hc<AppType>('http://localhost:3000');
const res = await client.users.$get();
```

> `AppType` is generated by running `npx zelt-hono-client generate`

### Programmatic Generation

When the build isn't driven by the Zelt CLI, call `GeneratorService.generateFromApp()` directly:

```typescript
import { GeneratorService } from '@zeltjs/hono-client';

const generator = new GeneratorService();
const content = await generator.generateFromApp(app.http, { distDir: './dist' });
```

Pass `portable: true` (with `tsconfig` and `projectRoot`) to emit a self-contained
type file with resolved literal types instead of imports into `distDir` — for
consumers that can't import from the server's `dist`, such as an Electron
renderer or a separate frontend workspace:

```typescript
const content = await generator.generateFromApp(app.http, {
  distDir: './dist',
  portable: true,
  tsconfig: './tsconfig.json',
  projectRoot: '.',
});
```
